Semantic Cache Management

How Semantic Caching works in this AI Gateway:

  • Requests with identical prompt structures generate a unique SHA-256 fingerprint: SHA-256(provider + model + messages).
  • When a match is found in Redis, the gateway immediately returns the cached ChatResponse in < 5ms without calling provider APIs.
  • This saves significant API budget and provides instant responses for repetitive client prompts.
